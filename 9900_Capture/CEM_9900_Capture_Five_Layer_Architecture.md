# Capture — Layered Evaluation Architecture (Five Layers)

**Date:** July 3, 2026
**Source:** CEM_Conversation_Discoveries_July_3_2026.md, item 13
**Conflict to resolve:** This proposes five layers. An earlier capture (`9900_Capture_Standardization_vs_Depth.md`, same date) already proposed three layers (Discovery, Evaluation, Decision). These have not yet been reconciled.

---

## The five-layer proposal

- Layer 1 — Company values
- Layer 2 — Role
- Layer 3 — Project / Team
- Layer 4 — Investigation
- Layer 5 — Human decision

## The earlier three-layer proposal (for comparison)

- Layer 1 — Evidence Discovery (how evidence is found; open-ended, candidate-led)
- Layer 2 — Evidence Evaluation (how each piece of evidence is judged; standardized per company)
- Layer 3 — Decision (how evaluated evidence is combined into a recommendation; company-defined weighting)

## Initial read on the relationship between them

These may not actually conflict — they may operate at different resolutions. The five-layer model looks like it could be describing *what shapes the evaluation lens* (company values → role → project/team, i.e., increasingly specific context that determines what "good evidence" even means for this decision), while the three-layer model describes *the mechanical process evidence goes through* (discovery → evaluation → decision) once that lens already exists.

If that reading holds, the five-layer model's first three layers (Company values, Role, Project/Team) might actually be inputs that configure the three-layer model's Layer 2 (Evidence Evaluation / the company's evaluation lens) — rather than being a competing structure.

This is a hypothesis, not a resolved conclusion. It needs its own real discussion before either model is drafted into a chapter.
